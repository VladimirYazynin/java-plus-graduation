# Explore With Me

**Explore With Me** - это бэкенд‑платформа для афиши событий, где пользователи могут находить интересные мероприятия, создавать собственные и участвовать в них.

---

## Архитектура проекта

Система построена как набор микросервисов, что обеспечивает гибкость, масштабируемость и устойчивость к отказам.

#### Инфраструктурные компоненты

* **Config Server** (`config-server`) - централизованное хранилище конфигураций микросервисов, позволяющее менять настройки без их перезапуска.
* **Discovery Server (Eureka)** (`discovery-server`) - реестр сервисов; каждый микросервис регистрируется в нём и находит другие сервисы по имени.
* **API Gateway** (`gateway-server`) - единая входная точка для клиентов, выполняющая маршрутизацию запросов и при необходимости аутентификацию, логирование и rate limiting.

#### Бизнес‑микросервисы

* **Event Service** (`event-service`) - управляет событиями, категориями и подборками, реализует основную доменную логику.
* **User Service** (`user-service`) - отвечает за жизненный цикл пользователей: создание, чтение, обновление, удаление.
* **Request Service** (`request-service`) - обрабатывает запросы на участие в событиях: создание, подтверждение, отклонение.
* **Comment Service** (`comment-service`) - управляет комментариями к событиям.
* **Statistics Service** (`stats-server`) - собирает и предоставляет статистику просмотров и обращений к событиям.

#### Хранение данных (Database per Service)

Каждый бизнес‑сервис использует свою часть базы данных **PostgreSQL**, что снижает связанность и упрощает эволюцию схем.

#### Взаимодействие сервисов

Межсервисное взаимодействие реализовано через REST. Для вызовов между сервисами используется декларативный клиент **Feign**, интегрированный с Eureka для динамического поиска экземпляров.

1. Внешний HTTP‑запрос поступает на API Gateway.
2. Gateway, опираясь на данные из Eureka, находит целевой сервис и проксирует запрос.
3. При необходимости `event-service` обращается к другим сервисам (например, к `user-service` за данными пользователя) через Feign‑клиенты, которые также используют Eureka для обнаружения.

## Основные возможности (API)

Все API доступны через API Gateway и разделены по ролям и сценариям использования.

#### Публичные (Public API)

* Поиск и просмотр событий с различными фильтрами и сортировками.
* Просмотр категорий и подборок событий.
* Получение списка комментариев к событию.

#### Приватные (Private API)

* Создание, редактирование и просмотр собственных событий.
* Создание, просмотр и отмена своих запросов на участие.
* Управление статусами заявок на участие в своих событиях.
* Добавление, изменение и удаление собственных комментариев.

#### Администраторские (Admin API)

* Управление пользователями: создание, просмотр, обновление, удаление.
* Управление категориями, событиями и подборками.
* Модерация событий и комментариев.

## Технологический стек

* **Язык**: Java 21
* **Фреймворк**: Spring Boot 3.x
* **Spring Cloud**:
    * *Spring Cloud Gateway* - реализация API‑шлюза.
    * *Spring Cloud Netflix Eureka* - сервис регистрации и обнаружения.
    * *Spring Cloud Config Server* - централизованное управление конфигурацией.
    * *Spring Cloud OpenFeign* - декларативные REST‑клиенты.
* **Сборка проекта**: Maven
* **Контейнеризация**: Docker, Docker Compose
* **База данных**: PostgreSQL (подход *Database per Service*).
* **ORM**: Spring Data JPA, Hibernate
* **Вспомогательные библиотеки**:
    * Lombok
    * MapStruct
